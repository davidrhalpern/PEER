# Contributing to PEER

Thanks for your interest in contributing.

## Scope and constraints
- Unclassified only. Do not include classified information.
- This repository currently prioritizes documentation, governance, and design artifacts before implementation.

## How to contribute
1. Open an Issue describing the change or suggestion.
2. Create a branch that references the Issue.
3. Submit a Pull Request.
4. Keep PRs small and focused on one logical change.

## Review expectations
- `main` is protected.
- PRs are required for all changes.
- Discussion should remain professional and constructive.

## Suggested contribution areas
- Clarifying governance rules and lifecycle assumptions
- Improving documentation structure and readability
- Adding diagrams or mockups (unclassified)
- Identifying edge cases and “failure modes” in delegation/accountability workflows
